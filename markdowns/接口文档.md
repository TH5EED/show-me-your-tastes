## 🔐 一、用户认证模块

### 1.1 用户注册

**接口**: `POST /api/auth/register`

**描述**: 用户注册账号

**请求参数**:
```json
{
  "username": "musiclover",
  "email": "user@example.com",
  "password": "Password123!",
  "nickname": "音乐爱好者"
}
```

**参数说明**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名（3-20字符） |
| email | string | 是 | 邮箱地址 |
| password | string | 是 | 密码（8-20字符，包含大小写+数字） |
| nickname | string | 否 | 昵称（默认为用户名） |

**成功响应 (201)**:
```json
{
  "code": 201,
  "message": "注册成功",
  "data": {
    "userId": "user_123456",
    "username": "musiclover",
    "nickname": "音乐爱好者",
    "email": "user@example.com",
    "createdAt": "2026-04-21T10:30:00Z"
  }
}
```

**错误码**:
- `400`: 参数验证失败
- `409`: 用户名或邮箱已存在

### 3.3 创建品味集

**接口**: `POST /api/taste-sets`

**描述**: 创建新的品味集（用于页面2）

**Headers**:
```
Authorization: Bearer <token>
Content-Type: application/json
```

**请求参数**:
```json
{
  "title": "我心中最佳单曲",
  "description": "这些单曲陪伴我度过了很多难忘的时光...",
  "isPublic": true,
  "items": [
    {
      "type": "song",
      "id": "song_001",
      "order": 1
    },
    {
      "type": "album",
      "id": "album_001",
      "order": 2
    },
    {
      "type": "artist",
      "id": "artist_001",
      "order": 3
    }
  ]
}
```

**参数说明**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | 是 | 品味集标题（1-50字符） |
| description | string | 否 | 描述（最多1000字符） |
| isPublic | boolean | 是 | 是否公开 |
| items | array | 是 | 作品列表（最多100个） |

**成功响应 (201)**:
```json
{
  "code": 201,
  "message": "创建成功",
  "data": {
    "id": "ts_123456",
    "title": "我心中最佳单曲",

    "itemsCount": 3,
    "coverUrl": "https://example.com/covers/ts_123456.jpg",
    "createdAt": "2026-04-21T10:30:00Z"
  }
}
```

**错误码**:
- `400`: 标题不能为空 / 作品数量超过100

## 📝 八、错误码汇总

| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| 400 | 请求参数错误 | 检查参数格式和必填项 |
| 401 | 未授权/未登录 | 跳转登录页面 |
| 403 | 无权限 | 提示用户无权限操作 |
| 404 | 资源不存在 | 检查ID是否正确 |
| 409 | 资源冲突 | 如已收藏、已点赞等 |
| 500 | 服务器错误 | 联系管理员 |
